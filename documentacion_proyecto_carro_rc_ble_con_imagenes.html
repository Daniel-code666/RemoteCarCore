1. Resumen del proyecto

El proyecto consistió en transformar un carro RC de juguete en un vehículo controlado desde celular mediante Bluetooth Low Energy. El control original fue reemplazado por una arquitectura basada en ESP32-C3, un servo para la dirección, un driver BTS7960 para el motor DC y una aplicación Android construida con Ionic, Angular y Capacitor.

El sistema permite enviar comandos de aceleración, reversa, dirección proporcional, freno/stop y modo de potencia Normal/Sport.

2. Objetivos alcanzados

  • Reemplazo del sistema de control original por un ESP32-C3.
  • Control BLE desde una APK Android.
  • Lectura y parseo de comandos enviados desde la aplicación.
  • Control proporcional de aceleración y reversa mediante PWM.
  • Control proporcional de dirección usando un servo SG90.
  • Integración del driver BTS7960 para manejar el motor DC original.
  • Configuración de alimentación con batería 2S y regulador LM2596 a 5.1 V.
  • Indicador visual de modo Sport usando el LED azul controlable del ESP32-C3.
  • Corrección de conflictos LEDC entre servo y motor.

4. Arquitectura general

La solución final quedó dividida en dos bloques principales: aplicación móvil y firmware del ESP32-C3.

Celular Android
  └── App Ionic/Angular/Capacitor
        └── Envía payload BLE
              ↓
ESP32-C3 Super Mini
  ├── Servidor BLE
  ├── Parser de comandos
  ├── Control dirección SG90
  ├── Control motor BTS7960
  └── LED indicador modo Sport

El ESP32-C3 recibe comandos en formato de texto simple, los parsea y aplica los valores a cada subsistema.

5. Sistema de alimentación

Se utilizó una batería principal de 7.4 V para alimentar el sistema. El motor recibe potencia desde la batería a través del BTS7960, mientras que el ESP32, el servo y la lógica del driver se alimentan desde el LM2596 ajustado a 5.1 V.

Batería 7.4 V
   ├── Switch general
   │     ├── B+ BTS7960
   │     └── IN+ LM2596
   │
   └── GND común
         ├── B- BTS7960
         └── IN- LM2596

LM2596 OUT+ 5.1 V
   ├── ESP32 5V/VIN
   ├── Servo SG90 rojo
   └── VCC / R_EN / L_EN BTS7960

LM2596 OUT-
   ├── ESP32 GND
   ├── Servo GND
   └── BTS7960 GND
Importante: La batería 2S no debe conectarse directamente al ESP32 ni al servo. El ESP32 y el servo deben recibir 5 V regulados.

Durante las pruebas se observó que al activar el servo el voltaje bajaba aproximadamente a 4.7 V. Se ajustó el LM2596 a 5.1 V para mejorar el margen de alimentación.

7. Comunicación BLE

El ESP32-C3 funciona como servidor BLE. La aplicación móvil se conecta como cliente y escribe comandos en una característica BLE.

Nombre BLEBLE_Remote_Car
Service UUID65215951-565f-4530-ae33-166245c668cf
Characteristic UUID3986086c-190f-4de8-8164-24abdf042717
PropiedadesWRITE y WRITE_NR

Formato de payload

<mode>,<throttle>,<steering>,<brake>
Campo Valores Descripción

throttle-100..100Reversa, neutro o avance
steering-100..100Izquierda, centro o derecha

Ejemplos

N,0,0,0       → modo normal, detenido, dirección centrada
S,80,-30,0    → modo sport, avance 80%, giro parcial
N,-40,25,0    → modo normal, reversa 40%, giro parcial
S,0,0,1       → modo sport, freno/stop

10. Control del motor

El motor DC original se controla mediante el BTS7960 usando dos señales PWM: una para avance y otra para reversa.

Constantes principales

constexpr int MOTOR_RPWM_PIN = 5;
constexpr int MOTOR_LPWM_PIN = 6;

constexpr int MOTOR_PWM_FREQ = 1000;
constexpr int MOTOR_PWM_RESOLUTION = 8;

constexpr int MOTOR_RPWM_CHANNEL = 4;
constexpr int MOTOR_LPWM_CHANNEL = 5;

constexpr int MOTOR_MAX_PWM = 100;
constexpr int MOTOR_SPORT_MAX_PWM = 110;
constexpr int MOTOR_BRAKE_PWM = 45;

Notas de implementación

  • Se usaron canales LEDC 4 y 5 para evitar conflicto con la librería del servo.
  • El modo Sport aumenta el PWM máximo permitido.
  • El modo Normal mantiene un límite de PWM más conservador.
  • El motor original no debe recibir toda la potencia de la batería 2S sin limitación.

Lógica conceptual

throttle > 0  → Forward(pwm)
throttle < 0  → Backward(pwm)
throttle = 0  → Stop()
brake = 1     → Brake() o Stop()
Nota: Si se usa MOTOR_MIN_PWM, debe aplicarse únicamente cuando throttle != 0. Para máxima estabilidad se puede mapear desde 0 hasta el máximo de modo.

11. Control de dirección

La dirección se implementó con un servo SG90. El valor de dirección de la app se transforma a un ángulo físico del servo.

Constantes finales conservadoras

constexpr int SERVO_CENTER = 90;

constexpr int APP_LEFT_SERVO_ANGLE = 110;
constexpr int APP_RIGHT_SERVO_ANGLE = 70;

Estos valores se eligieron para reducir esfuerzo mecánico y evitar que el servo quedara haciendo fuerza contra los topes de dirección.

Método de dirección

void SGCarDirection::SetDirection(int steering)
{
    int steeringConstraint = constrain(steering, -100, 100);

    if (abs(steeringConstraint) <= 3) {
        steeringConstraint = 0;
    }

    int servoAngle = SERVO_CENTER;

    if (steeringConstraint < 0) {
        servoAngle = map(
            steeringConstraint,
            -100,
            0,
            APP_LEFT_SERVO_ANGLE,
            SERVO_CENTER
        );
    } 
    else if (steeringConstraint > 0) {
        servoAngle = map(
            steeringConstraint,
            0,
            100,
            SERVO_CENTER,
            APP_RIGHT_SERVO_ANGLE
        );
    }

    steeringServo.write(servoAngle);
    lastServoAngle = servoAngle;
}
Se confirmó que cuando el sistema recibe steering = 0, el ESP32 ordena servoAngle = 90. Si físicamente las ruedas quedan desviadas, la causa probable es mecánica: tensión del varillaje, fricción, tope físico o falta de torque del SG90.

12. Indicador LED modo Sport

El modo Sport se indica con parpadeo del LED azul controlable de la ESP32-C3 Super Mini.

Hallazgo importante

El pin correcto no fue LED_BUILTIN, sino GPIO8. Además, el LED es activo en bajo.

int status_led_pin = 8;

Control del LED

void LedControl::SetStatusLed(bool on) {
    digitalWrite(status_led_pin, on ? LOW : HIGH);
}

void LedControl::UpdateSportLed() {
    if (currentMode != 'S')
    {
        sportLedState = false;
        SetStatusLed(false);
        return;
    }

    unsigned long now = millis();

    if (now - lastSportLedBlinkTime >= SPORT_LED_BLINK_MS) {
        lastSportLedBlinkTime = now;
        sportLedState = !sportLedState;
        SetStatusLed(sportLedState);
    }
}

13. Problemas encontrados y soluciones

Problema Causa identificada Solución aplicada
ESP32 conectaba/desconectaba por USB Problemas físicos de conector/cable/soldadura Nuevo ESP32-C3 soldado con más cuidado y ajuste del conector.
El motor no giraba R_EN y L_EN del BTS7960 sin 5 V Conectar R_EN y L_EN a 5.1 V desde LM2596.
Servo y motor se activaban al tiempo Conflicto de canales LEDC con ESP32Servo Usar canales 4 y 5 para el motor.
Comandos BLE llegaban con retraso Acumulación de escrituras BLE Uso de escritura sin respuesta y throttling de envíos.
Dirección invertida Orientación física del servo Invertir lógica o mapear ángulos desde el punto de vista de la app.
Dirección no simétrica Geometría mecánica del varillaje Usar ángulos independientes para izquierda y derecha.
Servo quedaba haciendo fuerza Recorrido excesivo o tope mecánico Reducir ángulos a valores conservadores.
LED Sport no parpadeaba Pin incorrecto y lógica invertida Usar GPIO8 y controlar LED como activo en bajo.
Caída de voltaje al mover servo Picos de consumo del SG90 con carga mecánica Ajustar LM2596 a 5.1 V y revisar alimentación directa desde regulador.

14. Recomendaciones de seguridad y mantenimiento

  • No conectar la batería 2S directamente al ESP32 ni al servo.
  • Verificar polaridad antes de energizar el sistema.
  • Mantener tierra común entre batería, BTS7960, LM2596, ESP32 y servo.
  • Evitar PWM excesivo en el motor original RC-260RA con batería 2S.
  • Revisar temperatura del motor, BTS7960, LM2596 y cables después de cada prueba.
  • Si el servo zumba o queda forzado, reducir recorrido o corregir mecánica.
  • No forzar la dirección contra topes físicos.
  • Usar cables adecuados para la corriente del motor y del servo.
  • Validar siempre el voltaje de salida del LM2596 antes de conectar el ESP32.
  • No cargar la batería sin cargador adecuado para su tipo y número de celdas.
Advertencia: Una batería 2S cargada puede entregar alrededor de 8.4 V. Ese voltaje puede dañar el ESP32 o el servo si se conecta sin regulación.
